<details>
<summary>📋 Guidance</summary>

The Zachman Diagram is a cross-cutting classification artifact that maps the engagement's
architecture content across the 6×6 Zachman Framework grid. It serves three purposes:

1. **Coverage tracking** — shows which cells have content, which are partial, and which are
   missing. Gaps indicate architecture concerns not yet addressed.

2. **Stakeholder communication** — each row targets a different audience (executive through
   implementer). Use this to select which cells to present to which stakeholder.

3. **Traceability** — each populated cell references the source artifact and section, so any
   stakeholder can navigate from the summary back to the detail.

Row 6 (Functioning Enterprise) is intentionally left as "Observed — not modelled in advance."
It represents the running enterprise, not an architecture specification.

Use `/ea-zachman generate` to auto-populate cells from existing engagement artifacts.
Use `/ea-zachman review` to produce a coverage matrix and gap analysis.
Use `/ea-zachman interview` to fill empty cells through guided Q&A.

**Cell notation:** R{row},C{column} — e.g. R2,C1 = Row 2 (Business Owner), Column 1 (What/Data).

</details>
