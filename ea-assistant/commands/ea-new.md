Create a new EA engagement project under `EA-projects/`.

## Instructions

1. If an engagement name was provided as an argument, use it as the display name. Otherwise, ask the user for:
   - **Name** (required) — display name of the engagement
   - **Description** (required) — brief description of the engagement
   - **Sponsor / Owner** (required) — name of the executive sponsor
   - **Organisation** (required) — organisation or business unit
   - **Scope** (required) — brief scope statement
   - **Engagement Type** (required) — Greenfield / Brownfield / Assessment-only / Migration
   - **Start Date** (required, default: today's date) — offer today's date as default
   - **Status** (default: `Active`) — Active / On Hold / Planning

2. Generate a slug from the name: lowercase, replace spaces with hyphens, remove special characters. Example: "Acme Retail Transformation 2026" → `acme-retail-transformation-2026`

3. Check that `EA-projects/{slug}/` does not already exist. If it does, inform the user and ask for a different name or whether to use the existing one.

4. Create the directory structure and seed files:
   ```
   EA-projects/{slug}/
   ├── requirements/
   ├── artifacts/
   ├── diagrams/
   ├── uploads/
   ├── interviews/
   ├── reviews/
   ├── ResearchAndReferences/
   │   └── research-index.md    ← seed with empty index
   └── brainstorm/
       └── brainstorm-notes.md    ← seed with heading and instructions
   ```

   Seed `ResearchAndReferences/research-index.md` with:
   ```markdown
   ---
   indexType: research
   engagementSlug: {slug}
   lastUpdated: {today ISO 8601}
   ---

   # Research & References — {name}

   Add research documents, notes, and links with `/ea-research`.

   | # | Type | Title | File | Added | Tags |
   |---|------|-------|------|-------|------|
   ```

   Seed `brainstorm/brainstorm-notes.md` with:
   ```markdown
   # Brainstorm Notes — {name}

   Use this file for freeform background, constraints, political context,
   stakeholder dynamics, and anything else that informs the engagement but
   doesn't belong in a structured artifact.

   This file is **never overwritten** by EA Assistant — it is yours to maintain.
   The EA interviewer surfaces relevant content from here during interviews.
   ```

5. Write `EA-projects/{slug}/engagement.json` with all fields populated and all ADM phases set to `Not Started`.

6. Read `.claude/ea-assistant.local.md` if it exists and extract `requirementsRepoPath`. Store in `engagement.json`. If the file does not exist, set `requirementsRepoPath` to `""`.

7. Write `EA-projects/{slug}/CLAUDE.md` using the **Project CLAUDE.md Template** below. This file is loaded automatically by Claude Code when a session is opened from inside the engagement folder, giving instant engagement context without needing to run `/ea-open` manually.

8. Confirm success to the user and display:
   - Engagement name and slug
   - Folder location: `EA-projects/{slug}/`
   - Offer to begin the **Preliminary phase** immediately or return to the main menu

## Project CLAUDE.md Template

Write this file to `EA-projects/{slug}/CLAUDE.md`, substituting all `{placeholders}` with actual values from the engagement. Where a section has no data yet (e.g. no Goals captured), write the section heading with "None captured yet." — never omit the section.

```markdown
# EA Engagement: {name}

<!-- Auto-generated by EA Assistant. Refreshed each time /ea-open is run. Do not edit manually — use brainstorm/brainstorm-notes.md for persistent notes. -->

## Engagement Identity

| Field | Value |
|-------|-------|
| **Name** | {name} |
| **Slug** | {slug} |
| **Type** | {engagementType} |
| **Organisation** | {organisation} |
| **Sponsor** | {sponsor} |
| **Scope** | {scope} |
| **Current Phase** | {currentPhase} |
| **Status** | {status} |
| **Domains** | {architectureDomains joined by ", "} |
| **Started** | {startDate} |
| **Last Modified** | {lastModified} |

## Strategic Intent

> This section is the primary context for all Claude sessions in this folder. It answers WHY this engagement exists and WHAT success looks like.

**Vision:** {direction.vision — or "Not captured yet"}

**Mission:** {direction.mission — or "Not captured yet"}

**Business Drivers:**
{for each DRV-NNN in direction.drivers: "- DRV-NNN: {statement}" — or "None captured yet."}

**Goals:**
{for each G-NNN in direction.goals: "- G-NNN: {statement} [{priority}]" — or "None captured yet."}

**Objectives:**
{for each OBJ-NNN in direction.objectives: "- OBJ-NNN: {statement} (target: {target} by {deadline})" — or "None captured yet."}

**Key Strategies:**
{for each STR-NNN in direction.strategies: "- STR-NNN: {statement}" — or "None captured yet."}

**Open Issues:**
{for each ISS-NNN in direction.issues: "- ISS-NNN: {statement} [threatens {linkedGoals}]" — or "None captured yet."}

## Artifact Status

{for each artifact in engagement.json artifacts[]: "- {name} ({phase}) — {status} / {reviewStatus}" — or "No artifacts yet."}

## Phase Progress

{for each phase in phases{}: "- {phase}: {status}" — only phases that are not "Not Started" or "Not Applicable"; if all are Not Started write "No phases started yet."}

## Open Decisions

{read each artifact file's Appendix A3 Decision Log; list all rows with state "Provisional" or "Awaiting" as: "- [{artifact}] {decision description} — {state} (owner: {owner})"}
{if none: "No open decisions."}

## Session Notes

Freeform context, background, constraints, and stakeholder notes that should not go into a structured artifact live in:

> `brainstorm/brainstorm-notes.md` — user-maintained, never overwritten

## Quick Commands

| Command | Description |
|---------|-------------|
| `/ea-open` | Load engagement with full details and next-action menu |
| `/ea-phase` | Continue the current phase ({currentPhase}) |
| `/ea-interview` | Run a stakeholder interview |
| `/ea-artifact` | Create or view artifacts |
| `/ea-grill [artifact]` | Deep-review an artifact using a grill-me skill |
| `/ea-decisions` | Generate Decision Register from all A3 logs |
| `/ea-research` | Manage research documents, notes, and links |
| `/ea-publish` | Produce consolidated architecture report |
```
